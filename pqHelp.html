Introduction

PPQT is a unified tool for use by the volunteers who post-process books for Distributed Proofreaders. It provides a text editor that integrates a number of features useful during post-processing, including display of the book's scanned pages, regular expression find/replace, spellcheck, pagination, footnoting, and html preview. This Help is its main documentation. Some additional info can be found in the Extras folder in the PPQT folder.

You can read this help text in the Help panel of the program or simply find the file pqHelp.html in the PPQT folder and open it in any browser.

While reading this text in the Help panel, if you click on a link, you can get back to the text by keying ctl-[ or ctl-b (mac users see next paragraph).

Mac Users Note

Throughout this Help, when you read ctl- you think cmd-. And when you read alt-, you think opt-. Qt is quite consistent in these mappings. However, when you see control- (not ctl-), that really means, the Control key on every platform.

For example, in this Help panel the "back" function is ctl-b or ctl-[. For a mac, that means cmd-[ or cmd-b. Similarly, it says below that to set bookmark 1, you key ctl-alt-1. As a Mac user you will know to substitute cmd- for ctl- and opt- for alt-, and set bookmark 1 using cmd-opt-1.

Files and Folders

PPQT edits one book text at a time, for example afinebook.txt. If you need to open a second document (for example, a commented copy from Smooth Reading), just start a second copy of PPQT.

File Encodings and Suffixes

PPQT holds the document in memory in Unicode, which supports virtually any alphabet or special characters. This is subject to two limitations:

  • PPQT can only display characters that are defined in the font you select (see Edit Panel below). DPCustomMono2, for example, lacks Greek and Cyrillic. Courier New, however, has a wide repertoire including Hebrew and Arabic.
  • Although it can display alphabets such as Arabic, PPQT does not understand right to left editing. The edit cursor only moves left to right during text entry.

The characters in documents on disk are encoded in some way. The important encodings are:

  • UTF-8, which can represent any Unicode character.
  • ISO-8859-1, aka Latin-1, a set of 191 characters.
  • Windows CP1252 contains the same 191 characters as Latin-1 but adds 27 more that are not in Latin-1. CP1252 is not supported by PG, but Windows utilities such as Wordpad may save files in that encoding.
  • Mac Roman defines many, but not all, of the characters in CP1252 (but with different numeric values), and also some special characters not in either CP1252 or Latin-1. Mac Roman is not supported by PG, but Mac utilities such as Textedit may save files in that encoding.

There are many other encodings but PPQT only supports the above four for input, and only UTF-8 and Latin-1 for output.

In order to load a document with its special characters intact and un-garbled, PPQT needs to know which encoding the file uses. It has several ways of finding this out.

  1. When saving a file, PPQT writes the encoding into the metadata file (next topic). Thus once you have saved a book it always reopens with the same encoding.
  2. Project Gutenberg (PG) requires all etext files to have the suffix .txt regardless of their encoding (crazy, but true). However it is conventional to indicate UTF-8 by ending the filename with -u, -utf, or -utf8, for example, afinebook-utf.txt. PPQT recognizes this and opens such a file as UTF-8.
  3. You can also end a filename with -l, -ltn or -ltn1 to signal that it is encoded Latin-1, for example myfinebook-ltn.txt.
  4. Per the standard, HTML files should either be UTF-8 or else they should have a charset=encoding parameter in the first 1024 characters of the file. When loading or saving a file of type .htm or .html, PPQT looks for this parameter and uses the encoding specified. If there is no parameter, it uses UTF-8.
  5. Although PG does not permit using file suffixes other than .txt for an etext, other suffixes are still useful for non-book files such as a good_words or "scannos" word list. Hence PPQT looks for these file suffixes:
    • .utf or .utx to signal UTF-8
    • .ltn to signal Latin-1
    • .win to signal Windows CP1252
    • .mac to signal Mac Roman
  6. When none of methods 1-5 reveal the encoding, PPQT feeds the first 4096 bytes of the file to a package that attempts to infer the encoding from the byte values used. If the package reports an encoding at a high confidence level PPQT uses that encoding. Otherwise it displays an error message.

You can bypass this complexity by using the File menu command Open With Encoding, and selecting one encoding from the sub-menu. The file you select is opened with that encoding.

When saving a file, PPQT will only use UTF-8 or Latin-1. It applies rules 2-5 above to discover which you want to use. To force one or the other, save the file with an indicator in the filename or file suffix, e.g. to force saving as Latin-1, save to bookname-ltn.txt.

Metadata

When opening afinebook.txt, PPQT looks for a file named afinebook.txt.meta in the same location (i.e. the full book name plus .meta). If the meta file is not found, PPQT assumes this is the first time it has opened this book. It scans the text gathering information including the location of page separator lines, and counting words and characters. These items and other useful things (defined bookmarks, your notes, the file encoding, etc.) are saved in the .meta file when the file is saved. When the .meta file does exist, PPQT loads the metadata items from it.

Feel free to examine or edit the metadata file, but be aware that it is encoded UTF-8. If strange characters appear, your editor app did not use the utf-encoding to load it. Do not save the metadata file unless you are sure you are loading and saving it in UTF-8 format.

Page Images

PPQT expects to find a folder named pngs in the same location as the text file, containing the scan images for the book. These are displayed in the Pngs panel.

The Main Window

When launched, PPQT displays a single window divided in two by a vertical bar. The left side is the Edit Panel where you edit the document. The right side is a tabbed set of panels for different functions. All these panels are described below.

You can adjust the size of the main window and its location on the screen. You can drag the vertical "splitter" left and right to give more or less space to each half. These geometry changes are remembered from session to session.

The Edit Panel

The left-side panel is a typical modern plain text line-editor. It uses a monospaced font, defaulting to Deja Vu Mono when available or to Courier New. Use the View > Font... command to select a font from your system, which will be remembered from session to session. It would be a very bad idea to use anything but a monospaced font.

Below the edit panel is a status area where activity messages appear from time to time. Bottom center is a line-number field displaying the line number of the edit cursor. Type a new number in that field and press Return to jump to a different line.

Next to the line number field is a field showing the cursor column position. When no text is selected, this is the column number of the character to the right of the insertion point. When there is a selection, the position number is either the left or the right end of the selected text, depending on how the selection was made (e.g. by dragging left to right, or right to left).

Further to the right is a progress bar that shows the status of some long-running operations.

Selecting Text

Select text as usual by dragging, or by click then shift-click, or by doubleclick (word) or tripleclick (line). Extend a selection by holding the shift key when using a cursor movement key or selecting a bookmark (see below). PPQT does not support rectangular selections or discontinuous selections.

You can move text by dragging the selection, and you can copy text by alt-dragging, in addition to the usual copy and paste keystrokes and Edit menu commands.

Edit Keys

The Edit panel supports the following special keystrokes.

Cursor Movement Keys

The arrow keys, Home, End, PageUp and PageDown keys work together with the control key to move the edit cursor following the usual conventions for your operating system. Add the shift key to any cursor move to extend the selection by that amount. For example, ctl-right arrow goes to the end of the line, but shift-ctl-right arrow selects text to the end of the line.

Editing Keys

  • ctl-c/ctl-x/ctl-v: copy/cut/paste
  • ctl-z: undo
  • ctl-y, also shift-ctl-z: redo
  • ctl-u, ctl-l, ctl-i: change selection to uppercase, lowercase, titlecase (Unicode-aware)

Find/Replace Keys

The following keys work in conjunction with the Find panel.

  • ctl-f: display the Find panel and put the focus there
  • shift-ctl-f: copy selection to Find panel and put focus there ("find this")
  • ctl-g: repeat the last search forward (same as Find Next)
  • shift-ctl-g: repeat last search backward (same as Find Prior)
  • ctl-=: replace selection (same as Find panel replace #1)
  • ctl-t: replace selection and repeat find forward
  • shift-ctl-t: replace selection and repeat find backward

Miscellaneous Keys

  • ctl-hyphen: display text 1pt smaller
  • ctl-+: display text 1pt larger
  • ctl-alt-1 through ctl-alt-9: set bookmark 1-9 to cursor (bookmarks are saved in the metadata file)
  • ctl-1 through ctl-9: jump to bookmark 1-9
  • ctl-shift-1 through ctl-shift-9: extend selection to bookmark 1-9. (Note to Mac users: cmd-shift-3 is "capture screen" and cmd-shift-4 is "capture selection". If you want to use these as bookmark 3 or 4 with selection, go to System Preferences, Keyboard, Keyboard shortcuts, click on Screen Shots, and disable these two keys.

Context Menu

Right-click (Mac: control-click) anywhere in the Edit pane to bring up a context menu with some edit commands. If the current selection is a single word, this context menu has an item that will enter that word in the good-words list; see Spellcheck. This cannot be undone (except by editing the meta file).

Scanno Highlighting

Use the File > Scannos menu command to load a file of scannos: a text file with one word per line (e.g. extras/en-common). (You can use any of the filename or suffix rules listed above to indicate the encoding of the scannos file.)

The path to the scannos file is remembered from session to session and it is re-loaded each time PPQT is restarted, if it can be found.

Use the View > Scannos menu command to turn on scanno highlighting. This puts a lilac background on any word that appears in the scannos file. Be patient when initiating scanno highlighting, it may tie up PPQT for several seconds in a large book.

Spellcheck

When PPQT opens a book for the first time (no .meta file exists), it looks in the same folder for a file whose name begins with good_words and loads its contents. It looks for a file whose name begins with bad_words also. You can use any of the filename or suffix rules listed above to indicate the encoding of these files.

These lists of "good" and "bad" words are saved in the metadata file thereafter. More "good" words can be added from the Words panel or with the context menu.

Use View > Dictionary to select a dictionary for the main language of the book. This choice of main dictionary is remembered from session to session. Dictionaries for several languages are included; more can be added by a procedure documented elsewhere. After changing the main dictionary, click Refresh in the Words panel to re-check all spelling.

When building or refreshing its metadata, PPQT checks all words for spelling. A "bad" word is assumed to be misspelt. A "good" word is assumed to be correct. Any word that is not in those lists is presented to the spell-checker for testing against the current dictionary.

Use the View > Spelling command to turn on red-underlining of misspelt words. Be patient, this may lock up PPQT for several seconds in a large book.

You can mark any word, phrase or section to be checked against an alternate dictionary with the nonstandard tag <sd dictag>...</sd>, for example

He whispered <sd fr_FR><i>Je t'aime</i></sd>
Obviously these nonstandard tags must be removed before the book is finalized. The use of a single alternate dictionary does not slow spellcheck much. However, the use of two or more alternate languages may slow spellcheck a lot.

The Pngs Panel

Click the Pngs tab to display the Pngs panel, which displays scan images loaded from the pngs folder. It normally displays the image that corresponds to the present edit cursor position.

When the focus is in the Pngs panel (click in it), ctl-hyphen zooms out and ctl-plus zooms in. The current zoom setting is remembered from session to session. If the image is larger than the panel, scroll bars appear. Use them to center the text. The scroll position is kept as pages change.

At the bottom of the panel is a slider with which you can adjust the zoom from 15% to 250%. Beside it, the button "To Width" causes the display to zoom to fill the width of the panel with the text. "To Height" zooms it to fill the height of the panel with the text. Both of these buttons work by scanning the image pixel by pixel to find the outermost dark spots. Many scan images have random blots and dots in the margins that make PPQT think the text is wider or taller than it looks to your eye.

You can use the PgUp and PgDn keys in the Pngs panel to flip backward and forward through the scan images. However, as soon as you move the text edit cursor, the Pngs panel snaps back to the image corresponding to the cursor.

The Notes Panel

Click the Notes tab. This panel is a simple plain-text editor to hold notes on the book in progress. Whatever contents you type here are saved in the .meta file and reloaded with the book.

When the focus is in the Notes panel, the alt-ctl-L key causes the current line number of the Edit panel cursor position to be entered, in curly braces: {1475}. (Note: under Ubuntu, alt-ctl-L may be the keyboard shortcut for Lock Screen. Use Ubuntu's Preferences > Keyboard Shortcuts to change this to, e.g., shift-alt-ctl-L.)

The alt-ctl-P key causes the current scan image number to be entered in square brackets: [214]. Use these keys to relate your notes to locations in the book, for example

oe lig near {1475}
big table on [214].

Place the cursor in or to the right of a {nnn} line number in the notes and key ctl-L. The Edit panel cursor jumps to that line. Place the cursor in or to the right of a [ppp] page number and key ctl-p. The Edit panel cursor jumps to the top of that page's text.

Key ctl-f for a simple Find dialog for searching in the notes. If text is selected, it is pre-loaded in the find-text field. The search starts at the cursor and wraps around at the end of the notes.

The Find Panel

The Find panel has controls for search and replace, including saved searches.

Searching and Replacing are done within top and bottom boundaries which are initially the beginning and end of the document.

Upon doing any search, keyboard focus returns to the Edit panel.

Find Controls

The top row of five checkboxes affect the search.

Respect CaseWhen checked, search is case-sensitive.
Whole WordWhen checked, normal searches only match whole words. Does not apply to regex searches (use \b in the expression)
In Sel'nWhen checked, search and replace occur within the span of text that is selected when you click the First or Last button.
RegexWhen checked, the Find string is treated as a regular expression.
GreedyWhen checked, a regular expression matches all it can; otherwise as little as it can.

The Find text field is below the checkboxes. Here enter the text pattern to look for. At the left is a popup menu containing the last 10 find values you explicitly typed in the field. (These are remembered from session to session.) The Find text field turns pink when Regex is checked and the syntax is not valid.

Below the text field are four buttons that perform searches:

NextSearch for the Find text beginning at the edit cursor and going toward the bottom boundary. Pressing Return in the Find text field is the same as clicking Next.
PriorSearch for the Find text beginning at the edit cursor and going back toward the top boundary.
FirstIf In Sel'n is checked, set the top and bottom of the current selection as the search boundaries; otherwise set to the whole document. Search for the Find text beginning at the top boundary and going forward.
LastIf In Sel'n is checked, set the top and bottom of the current selection as the search boundaries; otherwise set to the whole document. Search for the Find text beginning at the bottom boundary and going backward.

To simply search forward from the cursor position, press Return while in the Find text field, or click Next. To search backward, click Prior. To search from the top, use First. To search backward from the bottom, use Last. Search does not "wrap around" at the end of the document. When any search succeeds, the focus moves to the Edit panel.

When the focus is in the Edit panel, keying ctl-F makes the Find panel visible and puts the focus in it. Keying shift-ctl-F does the same but also copies the current text selection into the Find text field.

When the focus is in the Edit panel, keying ctl-G is the same as Next and shift-ctl-G is the same as Prior.

Replace Controls

There are three Replace fields. They are identical in use. Each is a text field to enter the replacement text. On the left of each field is a popup menu with the last 10 replacement strings you used out of that field. (These are remembered from session to session.) On the right of each field is a Repl button that causes the current edit selection to be replaced with the field contents.

Further right are three checkboxes. When &Next is checked, use of any Repl button is followed by the effect of the Next button. When &Prior is checked, use of any Repl button is followed by the effect of the Prior button.

When the focus is in the Edit panel, ctl-= has the effect of clicking Repl on the first replace field; ctl-t is like clicking Repl and then Next; and shift-ctl-t is like clicking Repl and then Prior.

When ALL! is checked, clicking any Repl button causes all instances of the Find text within the search boundaries to be looked up and replaced. The replace-all operation is a single undo/redo step. Before the replacement is done, a message is shown: "Replace nn occurrences of find-text with repl-text?" and you have the option of cancelling the operation.

To replace every occurrence of a string, but only within a portion of the document:

  • Put the text to find in the Find text field.
  • Put the replace text in a Replace field.
  • In the Edit panel, select the span of text where replacements are to happen (for example, drag to select the Table of Contents).
  • Set the In Sel'n switch on.
  • Click First (or Last) to establish the boundaries. Also jumps to and selects the first (or last) occurrence of the string.
  • Optionally, keep clicking Next (or Prior) to step through the span of text and verify all the locations to be changed.
  • Set the ALL! checkbox on.
  • Click Repl on the appropriate Replace field.
  • Note the count of replacements in the dialog and if they look right, click OK.
  • Inspect the result and if wrong, key ctl-Z (or Edit > Undo).

User-defined Buttons

Below the Replace controls is an array of 24 pushbuttons. Here you can store find/replace operations to be used again later. To store an operation, right-click (Mac users: control-click) a button. A message pops up asking you to give a short label for the button. When you do so and click OK, that label is assigned to the button, and the state of all the find/replace controls—all checkboxes and entry fields—is stored in that button. Whenever you want to repeat that operation, click the button. All the find/replace controls are returned to the stored settings. The button contents are remembered from session to session.

To clear the contents of a button, right-click it, clear the label field to empty, and click OK.

To save the button settings to a file, select File > Save Find Buttons. Provide the name and location for the saved file. PPQT uses its encoding rules 2-5 to select between UTF-8 and Latin-1 for this file. A suffix of .utf is the simplest way to make sure all special characters are preserved. In the saved file each non-empty button is represented in text form. (Feel free to edit it.)

To load button settings from a file, select File > Load Find Buttons and choose a file of saved buttons. The file's encoding can be indicated by the name or suffix, or will be assumed to be UTF-8. Any valid button definitions in the file are assigned to the buttons. Rather strict controls are applied, but no diagnostics are issued. An invalid button definition is just silently ignored.

The intent of saving and loading buttons is to be able to save and re-use complex search operations, and to exchange them with other users. See the Extras folder in the PPQT folder for some saved button definitions and a note on the format of a file of saved buttons.

Regex Syntax

The Regular Expression (regex) support provided by Qt4 is similar to, but has fewer features than, the Perl regexes used in guiguts. (Qt version 5, planned for 2013, has better support.) The following forms are used to match text.

non-special charactersmatch themselves
\cEscape for special characters: except as documented below, \c matches c. Use \\ to match a slash.
\n Matches between lines. \n\n matches an empty line.
\xxxxxmatches Unicode character with hexadecimal value xxxx
. (dot)Matches any one character including newline!
\d matches a decimal digit (Unicode-aware).
\D matches a non-digit (Unicode-aware).
\s matches any whitespace character including newline (Unicode-aware).
\S matches a non-whitespace character (Unicode-aware)
\w matches a word character (any Unicode letter, number, or Mark)
\Wmatches anything \w will not.
\n matches the n-th captured text string (same as perl/guiguts $n)
[chars] matches any one of the chars; may use ranges e.g. [A-Z1-9]
[^chars] matches any character except chars
\bmatches the zero-width boundary between a word and non-word character. \bon\b matches only the word "on".
\B matches what \b doesn't. \Bon\B will not match the word "on" but will match inside "tone".

Any of the above expressions can be or'd with stiles: alpha|beta means alpha or beta. Any of those expressions may be put in parentheses to capture the matching text for use in replacements. Any expression E can be quantified with these suffixes:

E? zero or one of E
E+ one or more of E
E* zero or more of E
E{n,m} from n to m of E, and n or m can be omitted: E{n,} at least n of E; E{,m} at most m of E

Non-capturing parentheses force a match but do not capture the matching text: (?:The color is)\s(red|green|blue) can only match a string that begins "The color is" but the first capture, \1, is what matched (red|green|blue).

Lookahead tests with (?=E) and (?!E) are supported: Fred(?=\n) matches "Fred" only at the end of a line; Fred(?!\n) only when not at end of line. However the lookbehind tests of Perl regexes are not supported.

Note: owing to some tricks PPQT plays with the Qt regex support, the caret and dollar codes, which usually match at beginning and end of a line, only match at top and bottom search boundary respectively. Use \n to match at beginning and end of line.

As usual, replacement text for a regex match may contain \i to refer to the ith captured (parenthesized) text. However, none of the special replacements supported by guiguts (such as \U...\E to uppercase the text) are available.

The Characters Panel

Click the Char tab to show the Characters panel. This panel displays a table of all characters seen in the document, in five columns:

  • Glyph: The character symbol.
  • Value: The hex value of the Unicode character.
  • Count: The number of times the character appears in the document.
  • Entity: The HTML/XML Entity code for this character.
  • Unicode category: The Unicode category of the character.

Click on the heading of any column to sort the table on that column. After you have edited the document, the table is out of date. Click the Refresh button to rebuild all metadata and make the counts accurate.

In the upper right is a popup menu with three choices:

  • All: show all characters
  • ¬ 7-bit: show only characters not in 7-bit ASCII
  • ¬ Latin-1: show only characters not in the Latin-1 set

Use this menu to easily find characters that are not allowed in a 7-bit file or a Latin-1 file, or to discover if the file needs to be saved as UTF-8.

If you double-click the character glyph or value, the character is copied into the Find text field and a First search is done so the first use of the character is highlighted in the document. Do this to search for the character in the document.

If you double-click on the Entity column, the glyph is copied to the Find text field while the Entity code is copied to the first Replace text. This makes it easy to replace a character with its Entity code.

The Words Panel

Click the Word tab to see the Words panel. This is a table of all words in the document, in three columns:

  • Word: text of the word
  • Count: the number of times the word appears
  • Features: a string of letters showing information about the word, for example Aa---X. These codes contain, from left to right:
    • A if the word contains an uppercase letter, else hyphen
    • a if the word contains a lowercase letter, else hyphen
    • 9 if the word contains a digit, else hyphen
    • h if the word contains a hyphen, else hyphen
    • p if the word contains an apostrophe, else hyphen
    • X if the word is in bad_words or fails spellcheck, else hyphen
Click on any column to sort on that column. Set the Respect Case checkbox to decide if the Word column sort is case-sensitive. When the table is large you can quickly locate words beginning with a particular letter. With the keyboard focus is in the Words panel and the table sorted ascending on the first column, key the initial letter you want. The table scrolls to show the first word beginning with that letter. The jump is case-sensitive when the Respect Case switch is checked; that is, use shift-E to find words beginning with uppercase E.

Double-click a word to copy that word into the paste buffer, load the word in the Find text, and do a First search to highlight the first use of the word in the document.

After the text is edited the table is out of date. Click Refresh to rebuild all metadata and make the table current.

Use the popup menu in the upper right to filter the table to these categories:

  • ALL: show all words
  • UPPERCASE: show only all-cap words
  • lowercase: show only lowercase words
  • mIxEdcase: show words having both upper and lowercase letters
  • Numbers: show only all-numeric words
  • Alnumeric: show words having both letters and digits
  • Hyphenated: show words containing a hyphen
  • Apostrophes: show words containing an apostrophe
  • Misspelt: show words that fail spellcheck

You can select one or more words by clicking, dragging, shift-clicking or ctl-clicking in the first column. When a word or words are selected, you can key shift-ctl-c to copy the selected words for pasting elsewhere.

Right-click (Mac users: control-click) on a word in the first column to open a context menu with these options:

  • Add to goodwords: Add all selected words to the good-words list and show them as correctly spelled.
  • Similar words: Filter the word list to show only words that match the clicked word when letter case, hyphens, and apostrophes are ignored.
  • First harmonic: Filter the word list to show only words that are within one edit of difference from the clicked word.
  • Second harmonic: Show only words that are within one or two edits of the clicked word.

Use Similar words to find inconsistent use of hyphens. Use First and Second Harmonic to find likely misspellings of a word. (For example, use Second Harmonic on the word Footnote to find some improperly-coded Footnote markups.) After viewing Similar or First or Second Harmonic, select All from the popup menu above the table to see all words again.

The Pages Panel

Click the Page tab to display the Pages panel. The body of this panel is a table showing the information extracted from the page separator lines the first time the file was opened. The information is retained in the metadata even after the text of the page separator lines has been deleted. The columns are:

  • Scan#: the number of the scanned image, usually three or four digits; this is the filename of the .png image file in the pngs folder.
  • Format: the display format for a visible folio: arabic or roman numerals.
  • Action: how to compute and display the folio for this page
  • Folio: the folio (book page number) for this scanned page
  • Proofers: the PGDP user-ids who proofed this page at its various stages

The table is always in scan image order. Double-click a cell in the Scan# column to make the Edit panel jump to the top of that page's text.

Double-click a cell in the Action column to set the folio action to one of:

  • Add 1: this page's folio is +1 over the preceding page's
  • Set @: set this page's folio to the number in column 4
  • Skip folio: show no visible page number for this image

Double-click a number in the folio column to set a specific numeric value to go with the Set @ action.

Double-click a cell in the Format column to set the format to one of Arabic, roman (lowercase) or ROMAN (uppercase).

After making one or more of these changes, click the Update button to update the table to reflect the changes. Using these controls you can get the folios of the file to exactly match the folios in the original book.

Inserting Folio Text

At the top of the panel is a text-entry field and an Insert button. Use these to insert a text pattern at the start of every page (except pages for which the Action is Skip folio). The text pattern may contain \n to insert a line-break, and it may contain %f to insert the folio number for that page. Insert any unique pattern that you can extend later with regular expression replacements, for example [=%f=]. The full pattern for HTML folios in the side margin is:

<span class='pgnum'><a id='Page_%f' name='Page_%f' href='#Page_%f'>[Pg&nbsp;%f]</a></span>\n

Inserting folio text is a single undo/redo operation.

The Reflow Panel

Click the Flow tab to display the Reflow panel. This panel has controls related to reflowing paragraphs and marked-up sections in the ASCII etext, and for HTML conversion.

Markup Codes

Use the following codes to mark sections for special treatment in the reflow of ASCII etext and during HTML conversion. During reflow, normal paragraphs are flowed within a maximum width, usually 72 characters. Other markup sections are supported as follows:

/U..U/Unsigned list: text is reflowed by paragraphs, with the first line of each paragraph indented 2 and other lines indented 4.
/P..P/Poetry:single lines are indented as given in the second row of controls. Line numbers are pushed to the right margin.
/R..R/Each line is aligned right.

ASCII Reflow

ASCII reflow involves arranging the text to fit in a maximum width and also formatting quotes, tables, and other special sections.

Setting Margins and Indents

The top row of controls set the default indents for block quotes. The First indent applies to the first line of a paragraph; Left to the other lines. The Right indent causes lines to be shorter than the maximum width by this amount.

The second row of controls set the indents for poetry, in which the First indent applies to every line; the Left indent is used only if a line is too long to fit in the maximum width and is folded.

At the left of the third row is a pair of Text width controls, one for the optimal (preferred) line length, and a second for the absolute maximum line of text. When reflowing paragraphs most lines will around the optimal length and none will exceed the maximum. Typical settings are 72 and 75. The maximum length is the default width for tables.

For Centered text, setting the "doc" button causes each line to be centered on the text width value. Setting the "self" button adjusts a centered section as far to the left as possible leaving a 2-space indent.

You can specify the First, Left and/or Right indent for any markup section by writing F:nn L:nn R:nn following the start of the markup. For example /Q F:8 L:6 starts a block-quote markup with those margins, overriding the First and Left margins set with the visible controls, but using the Right from the control.

You can write these explicit indent values on Quote, Poetry and List markups. You can also use R:nn on Right markup to set a right margin; and L:nn on no-reflow (/*) markup to set a left margin.

Reflowing

Select a portion of text and click Reflow Now: Selection, or use Reflow Now: Document to reflow the whole document. Reflow is a single undo/redo operation: perform it; inspect the result; use ctl-z to undo; adjust the markup and try again. During reflow, the markup codes are retained, and you can apply reflow multiple times without harm.

Before doing reflow you can set the switches on the left center to skip over special sections. For example set the skip-Tables checkbox so that /T markup is treated the same as /X and left alone.

Normally you reflow after removing <i>, <b>, and <sc> markup. However you can preview reflow while these are still in place by setting the controls on the right center. For example if <sc> markup will be converted to uppercase, set the control to treat this markup as 0 length. (This does not work for tables at this time. Set the skip button for tables until you have removed the i/b/sc markup from the text.)

Paragraphs of normal text and paragraphs in block quotes and lists are reflowed within the text margin settings. First, Left and Right margin settings are deducted from both numbers, so if the text widths are set to 72 and 74 (for example), and the Left and Right margins are 4 and 4, the effective margins are optimal 64 and maximum 66.

Paragraph text is reflowed within the effective margins using an optimal algorithm which gives more even line-breaks than the usual "greedy" algorithm.

Nesting Markups

To a limited degree you can nest the markups. For example you can nest a /P poetry section within a /Q block quote section or a /U list section. You can nest a /R right align section in poetry or a quote to provide a signature or letterhead line. Multiple levels of nesting (/R within /P within /Q) are possible. The First, Left and Right indents for a nested section are taken relative to the indents for its containing section.

Table Markup

PPQT treats tables as a special kind of reflow. You mark up the table text with /T...T/ and some other parameters. During reflow the table is formatted. If the result is not perfect, undo, adjust the markup, try again.

A table is a series of column values arranged in rows. The column values within each row are separated in one of two ways: by strings of two or more spaces, or with stile (|) characters.

A single-line table has one table row per text line. It is marked up with /T..T/ Here is a simple table with two rows and two columns.

/T
column 1    755.00
another row  2000.00
T/

A multi-line table has one or more text lines per logical row, and the rows are separated by blank lines. The markup starts with /TM. Here is a multiline table, a typical table of contents:

/TM
Introduction.--The Present Need   xiii

Chapter I. Applied Art    1

Chapter IV. The Nature and
Properties of Clay   29
T/

PPQT indents a table two spaces, an etext requirement that prevents an etext reader from reflowing formatted text. It indents more if the table is nested in another markup. By default the table is formatted to fill the maximum paragraph width set in the controls, or narrower when the table is nested in a markup with left and right indents.

By default column text is aligned left, and columns are given widths roughly in proportion to their contents. However, you can specify detailed controls on the opening markup line.

Specify properties of the whole table with T(properties) where the properties can be:

  • W:nn for the width of the table.
  • S:'|' (a stile in single quotes) to have the right side of the finished table be a column of stile characters.
  • T:'-' to have the top line of the table be a row of hyphens.

For example /TM T(W:50) begins a multi-line table with a width of 50. The width specified here can be wider than the text margin, and this is not checked.

Set the default properties for all columns with C(properties), where the properties can be:

  • W:nn for the minimum width of every column. (PPQT will give columns more width than this to fill the table width, or to fit the contents of the widest cell.)
  • A:R/L/C/D to align the cell contents left, right, centered or decimal.
  • B:'-' to have each cell have a line of hyphens under it.
  • S:'|' to have the left side of each column be a column of stiles.

You can set the width and alignment properties of a specific column n by writing n(properties). The properties you can give for a specific column are:

  • W:nn for the minimum width of that column.
  • A:R/L/C/D to align the cell contents left, right, centered or decimal.
  • S:' ' to have the left side of this column be blank.
Here is a fully-specified table before reflow:

/TM TABLE(TOP:'-' WIDTH:40 SIDE:'|') COL(B:'-' S:'|') 2(A:R W:12)
cell  cell   now is the time for all 
one   two    good parties to end     

row 2 cell  @  some more exciting     
                prose 
T/

and after reflow:

/TM TABLE(TOP:'-' WIDTH:40 SIDE:'|') COL(B:'-' S:'|') 2(A:R W:12)
  ----------------------------------------
  |cell one |    cell two|now is the time|
  |         |            |for all good   |
  |         |            |parties to end |
  ----------------------------------------
  |row 2    |           @|some more      |
  |cell     |            |exciting       |
  |prose    |            |               |
  ----------------------------------------
T/

You can omit the contents of empty cells on the right end of a row. However, you cannot leave blank a cell or cells on any line within a row, or PPQT will assign text to the wrong column (as in the example above where "prose" fell from column 3 to column 1 because columns 1 and 2 were blank). To fill in for a cell that is logically empty, use a single @ character on every line, as in the above example.

PPQT assigns horizontal space to columns using some brain-dead heuristics. To get exactly the spacing you want, specify the width of the table and of each column individually.

As suggested by the example, you can spell out the names of properties, for example WIDTH:50 instead of W:50. (Actually, only the initial letter is checked, so WATERMELON:50 works, too.)

Decimal alignment makes the decimal separators line up vertically in a column. When you specify A:D for a column, you can also specify the width in two parts, for example W:7.3 where the first number (7) says the minimum width for the integer part, and the second (3) sets the minimum width for the decimal and the fraction digits. You may also specify D:'x' to set a different decimal separator character x, for example in a European text where comma is the conventional decimal "point". Here is a simple table with decimal alignment before reflow:

/T TABLE(WIDTH:30) 1(A:R) 2(A:D W:8.4 D:',')
Persimmons   € 24,20
Marshmallows  € 124,80
T/

And after:

/T TABLE(WIDTH:30) 1(A:R) 2(A:D W:8.4 D:',')
    Persimmons        € 24,20 
  Marshmallows       € 124,80 
T/

Sadly, decimal alignment is not supported in HTML, and such cells are aligned left. Also, decimal alignment can produce odd results when applied to cells that contain text, especially in a multi-line table.

Getting a table to flow properly can be tedious. The first time you flow it, it is likely to have many errors, often because there are extra spaces that cause false column breaks, or missing spaces that cause column values to run together. But with work it is usually possible to get a table to look right. Post a sample of a recalcitrant table to the forum and ask for advice.

HTML conversion is a single undo/redo operation: apply it; check the result in the Preview panel; key ctl-z to undo and fix the markup.

The Preview Panel

Click the Pvw tab to display the HTML Preview panel. Whenever you click either Refresh button on this panel, the complete contents of the Edit document are copied into this panel and displayed as HTML.

The default font is set to Palatino and the initial text font size is set to 16 points. When the focus is in the Preview panel, you can key ctl-minus and ctl-plus to zoom to smaller and larger text font sizes.

The base URL for image references is the base path of the book text file, so the images folder should be located at the same place as the book text.

If you click a link in the preview (for example, to open a high-resolution image) the browser follows the link and displays the linked page or image. To return to the book preview, key ctl-B or ctl-[ (control-left bracket).

The Refresh button only refreshes the HTML. When you click Refresh + Clear, the browser's memory cache is cleared and any images are reloaded from disk. Use this button if you make changes in the images.

The Footnote Panel

Click the Fnote tab to display the Footnote management panel.

Definitions

The following terms are important. A Note is a section of text that begins with [Footnote and ends with a right-bracket, for example,

[Footnote z: ... some text possibly mutiple lines,
ending with -- ]

A Note is only recognized when [Footnote begins in column zero and the final right-bracket is the last character on its line. (Additional square brackets within a Note are not a problem so long as they don't come at the end of a line.) A Note may contain other types of markup, for example quotes, poems or tables.

An Anchor is a reference to a Note enclosed in brackets thus[z]. An Anchor is never found in column zero, although it may come at the end of a line. Anchors may appear in tables, quotes, and poetry as well as normal text.

A Note should not be coded inside /Q or /T or any other markup section that gets indented, because after reflow the [Footnote will no longer be at the left margin, and the note won't be recognized. When an Anchor appears in some markup section, place the matching Note after the end of the markup.

A Note may contain an Anchor to another Note (that is, a footnote to the footnote—yes, it really happens), but the Note bodies cannot be nested. The referenced Note to the Note must be outside the referring Note, perhaps immediately after it.

A Key is the bit of text that relates an Anchor to its Note, z in these examples. A key falls into one of several Classes:

  • The letters IVXLCD are in the Upper Roman class
  • The letters ABCDEFGHIJKLMNOPQRSTUVWXYZ are Upper Alpha Class
  • The letters ivxlcd are Lower Roman class
  • The letters abcdefghijklmnopqrstuvwxyz are Lower Alpha Class
  • The digits 0123456789 are Decimal Class
  • The six symbols *¤§¶†‡ (asterisk, currency, section, paragraph, dagger and double-dagger) are Symbol Class

Only the Latin alphabet, digits, and the symbols shown are recognized as Key values. (Most PGDP books are proofed using only the letters A, B and occasionally C; some scholarly books might have lowercase roman keys or decimal keys as well.)

The alphabetic classes are ambiguous; that is, a letter Key might be either an alpha or a roman, depending on how you want your book to work. This is resolved when renumbering, see below.

The Footnote Table

When you click the Refresh button at the top of the panel, PPQT scans the document for Anchors and Notes. It matches the anchors and notes and displays the result in the table. The columns of the table are:

  • Key: the letters or numbers of the key.
  • Class: the Key Class that PPQT has assigned based on the Key.
  • Ref Line: the line number where the Anchor was seen.
  • Note Line: the first line number of the Note.
  • Length: the number of lines in the Note.
  • Text: the first few characters of the Note text.

When PPQT finds an Anchor but cannot match it to a Note, the Note Line column has a question mark. When it found a Note but no Anchor refers to it, the Ref Line column has a question mark. In these unmatched cases, the row of the table is pink to show there is a problem. You must resolve these problems before the action buttons at the bottom of the panel will work.

When a Note has more than 10 text lines, or an Anchor is more than 50 lines from its Note, the table row is pale green to suggest a possible problem. You should check these cases to make sure PPQT has recognized the Note properly and matched it to the correct Anchor.

When you click on a Ref Line cell, the document jumps to the line with that Anchor, with the Key selected. Click on a Note Line cell; the document jumps to that Note with the Note selected. Click repeatedly on a Key cell and the document jumps back and forth between the Anchor and the Note.

Renumbering

When you click the Renumber button, provided there are no pink lines in the table, PPQT sweeps through all the Keys in the document and renumbers them sequentially according to their Class. PPQT has five counters that are set to zero when you click Renumber, and that are increased by 1 each time they are used. The number streams are:

  • 1,2,..999 produces a series of decimal numbers.
  • A,B,..ZZ produces a series of uppercase letters.
  • a,b,..zz produces a series of lowercase letters.
  • I,II,..M produces a series of uppercase Roman numerals.
  • i,ii,..m produces a series of lowercase Roman numerals.

Below the table are six popup menus that let you specify how the Keys of each Class are to be renumbered. If the book has only uppercase alpha keys (the usual situation) then set Classes ABC and IVX both to use the A,B..ZZ stream. However, if the book has a separate set of lowercase Roman keys, set that Class to use the i,ii,..m stream. You could also use renumbering to change the Class of keys in use. If the book has only uppercase alpha, you could set that Class to use the 1,2,..999 stream; after renumbering, those keys will be Decimal class.

Renumbering is a single-undo operation. After renumbering, examine the results; if they are not satisfactory, click in the text and key ctl-z.

Footnote Sections and Moving Footnotes

Proofers leave Notes on the page where they appear. You move them to follow the paragraph that references them. However for some books you want to move all Notes to a special section at the end of each chapter, or at the end of the book.

To do this, you must edit the book to show where the footnotes are to go using the markup codes /F and F/. For example, a section of notes at the end of a chapter might be coded:

...last line of chapter text.


NOTES


/F
F/

Here, NOTES represents a subhead (because it is preceded and followed by two blank lines). The text "NOTES" does not matter; it is the /F..F/ markup codes that tell where the Notes from the preceding chapter should be moved. Alternatively you might have a single chapter of notes at the end of the book text.

When you click the Move Notes button, PPQT scans the document and locates all the /F..F/ sections you have coded. (If it cannot find any, it displays an error.) Then it goes through the footnote table and finds any Notes that are not already enclosed in a /F..F/ section. It moves these into the Footnote section next below them in the document. (If there is no section below a Note, it is not moved.)

Moving notes is a single-undo operation. Inspect the result and use ctl-z if it is not satisfactory.

ASCII Conversion

Click ASCII Cvt in the Footnote Panel to convert all Notes to block-quote /Q..Q/ markup. Each note is changed from:

[Footnote key: Text of the note... ]
into:
/Q F:2 L:w R:2
  [key] Text of the note...
Q/
The left-indent w is based on the widest key of that key class. These new /Q..Q/ sections can be reflowed in the Reflow Panel ASCII reflow.

Do this as the very last footnote operation in the plain text fork of the book. Afterward, the Footnote Panel is of no use because the Notes can no longer be recognized.

HTML Conversion

HTML conversion of footnotes is divided between the Footnote Panel and the Reflow Panel.

In the Footnote panel click HTML Cvt to convert all Anchors and Notes to HTML. This is a single-undo operation. Do this as the very last footnote operation in the HTML fork of the book. Afterward, the Footnote Panel is of no use, because the Notes can no longer be recognized.

This does not convert the the markup tags /F..F/. Later, when you use HTML Convert in the Reflow Panel, it converts the tags to <div> markup.
